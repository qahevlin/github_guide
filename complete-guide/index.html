Complete GitHub Command Guide

From account setup to advanced workflows, this page lists exact commands for each scenario. Designed for beginners through experts, all commands are copy-paste ready for terminal use.

1. Preflight Setup

Install Git and configure your identity:
sudo apt update && sudo apt install git -y (Linux) or install via downloaded package on Windows/macOS.
git --version
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global core.editor "code --wait" (use VS Code)
git config --global push.default simple
git config --global credential.helper cache

2. Create or Clone a Repository

A. New repo flow:
mkdir my-project && cd my-project
git init
echo "# My Project" > README.md
git add README.md
git commit -m "chore: initial commit"
git branch -M main
git remote add origin git@github.com:USER/REPO.git
git push -u origin main B. Existing repo flow:
git clone git@github.com:USER/REPO.git
cd REPO
git remote -v

3. Daily Development Workflow

A. Branching and coding:
git checkout -b feature/your-feature
git status
git add file1 file2 or git add .
git commit -m "feat: add ..."
git push -u origin feature/your-feature B. Sync with remote:
git checkout main
git fetch origin
git pull origin main --rebase
git checkout feature/your-feature
git rebase main (or git merge main)
If conflict, edit files, then git add . and git rebase --continue

4. Collaboration and PR Workflow

A. Push branch and open PR:
git push --set-upstream origin feature/your-feature
Then create PR at GitHub with matching branch and description. B. Update a branch after review comments:
git checkout feature/your-feature
Make code changes, then git add ., git commit --amend --no-edit or git commit -m "fix: ...".
git push --force-with-lease C. Merge PR through GitHub UI or CLI:
gh pr merge --merge (requires GitHub CLI)
gh pr merge --squash
gh pr merge --rebase

5. Merge Conflicts

Detect: git status
Resolve: edit files manually, then git add path/to/file
git rebase --continue or git merge --continue
git rebase --abort / git merge --abort to fallback

6. Recovery & History Management

Undo local edit before staging: git restore Undo staged file: git restore --staged Undo last commit (keep changes): git reset --soft HEAD~1 Discard last commit: git reset --hard HEAD~1 Drop unreferenced commits: git reflog, git checkout HEAD@{2}

7. Advanced Scenarios

Interactive staging: git add -p Stash work: git stash push -m "WIP", git stash list, git stash pop Cherry-pick commit: git cherry-pick Bisect errors: git bisect start / git bisect bad / git bisect good Squash commits: git rebase -i HEAD~N, then squash in editor.

8. Tagging and Release

Tag version: git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
git push origin --tags

9. Remotes and Forks

git remote -v
git remote add upstream git@github.com:ORIGINAL/REPO.git
git fetch upstream
git checkout main
git merge upstream/main
git push origin main

10. GitHub CLI Quick Calls

gh auth login
gh repo clone USER/REPO
gh issue list --label bug
gh pr create --fill --base main --head feature/your-feature
gh pr checkout
gh pr review --approve
